[2026-07-14] Verified AnimationManager loads all 11 real assets without
             error, then ran the actual app end-to-end (scripted, skipping
             the 45-min wait) and screenshotted each state. Found and fixed
             two real rendering bugs surfaced only by using real assets:

             BUG 1 - magenta fringe around the pet sprite. Root cause:
             animations.py resized frames to 220x220 with PIL's default
             bicubic filter, which blends the alpha channel at sprite edges;
             any partial-alpha edge pixel is not exactly the window's
             -transparentcolor key (#ff00ff) so Windows' colorkey
             transparency can't strip it, leaving a visible rim. Fix:
             animations.py now resizes with Image.NEAREST (no alpha
             blending) - appropriate anyway for pixel-art sprites.

             BUG 2 - Yes / "Remind Me In 5 Min" button text invisible
             (buttons rendered as flat color bars, no label). Root cause:
             NOT a customtkinter bug (reproduced identically with a plain
             tk.Button, and with customtkinter 6.0.0 and 5.2.2 both) - it's
             a Tk geometry-manager quirk. ui.py's hide_bubble()/show_bubble()
             used bubble_frame.pack_forget()/pack() to toggle the speech
             bubble (which contains the buttons). Whenever that pack_forget/
             pack cycle happened AND a sibling widget in the same pack-
             managed container (image_label) had its image changed via
             .configure(image=...), the re-shown frame's button text canvas
             came back blank. Confirmed via ~15 isolated repro scripts
             (see scratchpad, not kept in this repo) narrowing it down to
             exactly this combination. Fix: ui.py now positions bubble_frame
             and image_label with place()/place_forget() instead of pack()/
             pack_forget() at the container level, which avoids the sibling
             relayout cascade entirely. button_frame's own pack_forget()/
             pack() (show/hide the Yes/Snooze buttons within the bubble) was
             left as-is since it has no image-holding sibling and tested
             fine.
